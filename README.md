# Микросервисная система управления дефектами

## Настройка окружения

Создайте файл `.env` из примера:

```bash
copy env.example .env
```

Отредактируйте `.env` и измените `SECRET_KEY` на безопасный ключ.

## Запуск

```bash
docker compose up --build
```

## Остановка

```bash
docker compose down
```

## Структура сервисов

- **API Gateway** (порт 8000) - точка входа для всех запросов
- **Auth Service** (8001) - авторизация и пользователи
- **Projects Service** (8002) - управление проектами
- **Defects Service** (8003) - дефекты, комментарии, вложения
- **Reports Service** (8004) - отчёты и аналитика
- **Frontend** (порт 3000) - веб-интерфейс

## Доступ

- Frontend: http://localhost:3000
- API Gateway: http://localhost:8000
- Swagger: http://localhost:8000/docs

## Функциональность

- ✅ Регистрация и авторизация пользователей
- ✅ Управление проектами (CRUD)
- ✅ Управление дефектами (CRUD)
- ✅ Комментарии к дефектам
- ✅ Загрузка и управление вложениями
- ✅ Экспорт дефектов в CSV/Excel
- ✅ Аналитика и отчёты
- ✅ Управление ролями пользователей
